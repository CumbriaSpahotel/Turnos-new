PROMPT – REGLAS MAESTRAS DE PROTECCIÓN DEL SISTEMA “TURNOSWEB / HOTELES G+C”

Objetivo:
Blindar el sistema para que ningún desarrollador, agente automático, IA, script o refactorización modifique partes críticas sin autorización expresa.

Estas reglas son obligatorias. No son recomendaciones.

==================================================
0. PRINCIPIO GENERAL
==================================================

Nadie puede modificar lógica crítica, base de datos, identificadores, publicaciones, importadores o arquitectura sin autorización expresa.

Antes de tocar cualquier parte sensible:

1. Explicar qué se va a modificar.
2. Justificar por qué es necesario.
3. Indicar archivos/tablas afectadas.
4. Indicar riesgos.
5. Hacer backup si hay escritura.
6. Hacer dry run si hay importación o migración.
7. Esperar autorización expresa.

Frase obligatoria antes de cambios sensibles:

“Esta acción puede afectar datos operativos o históricos. Solicito autorización expresa antes de ejecutarla.”

==================================================
1. IDENTIDAD DE EMPLEADOS – PROTEGIDO
==================================================

PROHIBIDO sin autorización expresa:

- Cambiar `empleados.id`.
- Cambiar la primary key de empleados.
- Migrar definitivamente a UUID.
- Borrar empleados.
- Fusionar empleados.
- Renombrar IDs legacy.
- Reasignar `id_interno`.
- Reutilizar un EMP-XXXX dado de baja.
- Fusionar Natalio/Natalia.
- Fusionar Sergio/Sergio Sánchez.
- Eliminar el placeholder “¿?” sin confirmación.
- Cambiar el significado de `id_interno`.
- Usar nombres como identificador nuevo.

Regla protegida:

Empleado = identidad estable.
Nombre = solo visual.
EMP-XXXX = identificador administrativo visible.
UUID = identidad técnica paralela o futura.
ID legacy = compatibilidad histórica.

No modificar nada de identidad sin plan de migración aprobado.

==================================================
2. TURNOS BASE – PROTEGIDO
==================================================

La tabla `turnos` solo puede contener turnos base.

Valores permitidos:

- M
- T
- N
- D
- -

PROHIBIDO guardar en `turnos`:

- V
- VAC
- CT
- CAMBIO
- BAJA
- IT
- PERMISO
- PERM
- FORMACION
- SUSTITUCION
- texto largo tipo “Vacaciones”
- texto largo tipo “Intercambio turno”

Regla protegida:

`turnos` = base operativa limpia.

No modificar masivamente `turnos` sin:
- backup
- dry run
- informe de diferencias
- autorización expresa

==================================================
3. EVENTOS / INCIDENCIAS – PROTEGIDO
==================================================

Vacaciones, bajas, permisos, sustituciones y cambios NO son turnos base.

Deben vivir en `eventos_cuadrante` o módulo equivalente.

PROHIBIDO sin autorización expresa:

- Borrar eventos históricos.
- Cambiar tipos de eventos masivamente.
- Convertir eventos en turnos base.
- Mover sustituciones a `turnos`.
- Eliminar payload histórico.
- Cambiar empleado_id / empleado_destino_id / sustituto_id sin mapa.
- Modificar vacaciones ya cargadas.
- Modificar bajas/permisos ya cargados.
- Modificar cambios de turno ya cargados.

Regla protegida:

Eventos = incidencias.
Turnos = base.
Motor = resolución.
